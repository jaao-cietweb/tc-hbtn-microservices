Arquitetura Serveless trata-se de uma abordagem onde o código é executado de maneira difundida entre servidores que o serviço de cloud computing seleciona. De mãos dada a Serveless temos o serviço de nuvem conhecido como FaaS, ele se trata de um modelo de execução orientado a eventos que executa seus containers stateless.
Esta arquitetura vem para sanar alguns problemas na hora de implementar sua aplicação, estes problemas são: Custo operacional, quando se utiliza servidores na nuvem o custo vem apenas das chamadas de função realizadas, todo o custo que seria voltado a servidores presenciais é cortado. Além também de problemas como arquitetura serem reduzidos devido a utilização de arquitetura sem servidor, dessa forma operações de tradução de aplicação consomem menos tempo. Entre outros pontos positovos.
O cloud computing é deixar processos/funções como armazenamento, banco de dados, softwares entre outros na internet no serviço de nuvem, desta forma é mais fácil acrecentar uma nova funcionalidade e escalar recursos. Pagando apenas pelos serviços usados.
Arquitetura monolitica utiliza uma estrutura simples onde todo o projeto é acoplado e compatilham o mesmo banco de dados. Microserviços expande esse conceito e coloca várias funções com baixo acoplamento e bancos separados. Em Serveless temos um grande semelhança com MSA, porém ela é desenvolvida para não haver hardware ócioso.
